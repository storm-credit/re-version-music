# Re:Version Music

감정 재해석 음악 채널 프로젝트. 원곡의 감정 구조를 추출해 시점·문화·장르를 변환한 오리지널 곡을 만든다.

## 채널 컨셉

"같은 감정, 다른 버전" — 커버가 아닌 감정 번안·재해석 채널

## 콘텐츠 유형 (4가지)

1. **번안형** (40%) — 팝송 감성을 한국식으로, 또는 한국곡을 해외 감성으로
2. **답가형** (30%) — 원곡 화자의 상대방이 답하는 곡
3. **팬픽OST형** (20%) — 가상 캐릭터/세계관 테마곡
4. **오리지널형** (10%) — 완전 오리지널 실험곡

## 제작 파이프라인

0. 오케스트라 인테이크: 목표, 유형, 영감 원천, 산출물, 위험 요소 정리
1. 원형 추출: 화자, 상대, 사건, 핵심 감정
2. 변환 규칙: 언어/문화/시점/장르 변환 방향 결정
3. 후렴 우선 제작: 쇼츠용 한 방 후렴
4. 저작권·감정 일관성 1차 리뷰
5. 가사 작성: 1절 → 2절 → 브릿지
6. Suno 프롬프트 작성 및 생성
7. 영상 패키징: 상황형 영상 + 자막
8. 발행 후 성과 기록 및 다음 액션 정리

## 오케스트라 총괄 원칙

오케스트라는 직접 모든 결과물을 만드는 역할이 아니라, 전문가 산출물을 통합하고 승인/보류를 판단하는 총괄 디렉터다.

- 감정 원형, 변환 규칙, 후렴 훅, 저작권 안전선을 먼저 확인한다.
- 캐릭터는 곡마다 달라질 수 있지만, 감정 변환 문법은 일관되게 유지한다.
- 팬픽OST형만 캐릭터/세계관 일관성을 강하게 보고, 번안형·답가형은 시점과 문화 감정 번역을 우선한다.
- 작업마다 필요한 전문가만 배정한다: 감정 원형 분석가, 변환 전략가, 후렴/훅 작가, Suno 음악감독, 비주얼 감독, 저작권/브랜드 리뷰어, 발행/성과 분석가, 문서 아카이비스트.
- 최종 판단은 `승인 / 조건부 승인 / 보류`로 남기고 다음 액션을 기록한다.

## 딥 오케스트라 모드

아래 작업은 `오케스트라 회의록 템플릿`을 사용해 전문가별로 깊게 검토한다.

- 새 시리즈 또는 새 포맷을 시작할 때
- 원곡/IP명이 공개 제목, 설명, 썸네일에 들어갈 때
- 팬픽OST형처럼 캐릭터/세계관 일관성이 중요한 때
- Suno 프롬프트와 영상 프롬프트를 동시에 만들 때
- 발행 전 저작권 위험이 중간 이상일 때

딥 모드는 총괄 브리핑 → 감정 원형 → 변환 전략 → 후렴/훅 → Suno 음악 → 비주얼 → 저작권/브랜드 → 발행/성과 → 문서 아카이브 순서로 진행한다.

전문가별 호출은 `전문가 실행 프롬프트 카드.md`를 따른다. 최종 승인 판단은 `딥 오케스트라 승인 매트릭스.md`로 점수화하고, 강제 보류 조건이 있으면 총점과 무관하게 보류한다.

오케스트라 체계 자체를 세우거나 운영 리듬을 잡는 요청은 `오케스트라 총괄 수립 계획.md`를 기준으로 진행한다. 주간 점검이나 반복 운영은 `오케스트라 주간 운영 리포트 템플릿.md`로 기록한다.
