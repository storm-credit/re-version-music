# Re:Version Music 영상 제작 마스터 가이드

## 제작 파이프라인

```
1. Claude (곡 기획 + 가사 + 프롬프트)
   ↓
2. Suno (음악 생성, 4분 풀버전)
   ↓
3. 힉스필드/Midjourney (장면별 이미지 6~8장)
   ↓
4. CapCut (최종 영상 편집)
   ↓
5. 업로드 (YouTube Shorts + TikTok + YouTube 롱폼)
```

---

## 1단계: Suno 음악 생성

### 설정값
- **버전**: Suno v4
- **길이**: 4분 (Full Song)
- **모드**: Custom Mode
- **입력**: Style of Music + 가사 (섹션 태그 포함)

### Suno 섹션 태그 규칙
```
[Intro] — 4~8마디, 분위기 세팅
[Verse 1] — 스토리 시작
[Pre-Chorus] — 감정 빌드업
[Chorus] — 핵심 후렴 (쇼츠 하이라이트)
[Verse 2] — 스토리 전개
[Bridge] — 감정 전환점 / 클라이막스
[Final Chorus] — 감정 최고조 (전조 or 풀밴드)
[Outro] — 여운
```

### Suno 팁
- Style of Music에 **5~8개 키워드** 넣기 (장르, 악기, 보컬, 무드)
- 괄호 안 지시 `(piano solo, 8 bars)` → Suno가 반영할 수도 있고 안 할 수도
- 같은 프롬프트로 **3~5회 생성** 후 베스트 선택
- 마음에 안 드는 부분은 **Extend**로 재생성
- mp3 다운로드 후 `03. 제작/` 폴더에 저장

## 2단계: 이미지 생성

### 장면 구성 (곡당 6~8장)
```
Scene 1: 인트로/분위기 세팅 (넓은 풍경 or 상징물)
Scene 2: Verse 1 핵심 장면
Scene 3: Verse 2 핵심 장면
Scene 4: Chorus 감정 이미지 (강렬한 비주얼)
Scene 5: Bridge 전환점 (분위기 반전)
Scene 6: Final Chorus 클라이막스
Scene 7: Outro 여운 (페이드아웃 느낌)
Scene 8: 썸네일용 (텍스트 공간 확보)
```

### 힉스필드 프롬프트 구조
```
[Subject]: 구체적 장면 묘사
[Style]: Cinematic / Anime / Realistic / Illustration
[Mood]: 감정 톤
[Color]: 색감 지정
[Composition]: Wide shot / Close-up / POV / Aerial
[Details]: 소품, 조명, 시간대
```

### 이미지 규격
- **쇼츠용**: 1080×1920 (9:16 세로)
- **롱폼용**: 1920×1080 (16:9 가로)
- **썸네일**: 1280×720 (텍스트 영역 확보)

### 스타일 가이드 (시리즈별)
| 시리즈 | 스타일 | 색감 |
|--------|--------|------|
| BTS 답가 | 시네마틱 + 감성 | 보라+골드 |
| 번안형 | 문화별 차이 시각화 | 따뜻한 대비 |
| 팬픽OST | 일러스트/애니 | 세계관 맞춤 |

## 3단계: CapCut 편집

### 프로젝트 설정
- 해상도: 1080×1920 (쇼츠)
- FPS: 30
- 배경: 검정 or 이미지

### 편집 레이어 구성
```
Layer 1: 배경 이미지 (장면별 전환)
Layer 2: 카메라 효과 (줌인/아웃, 패닝)
Layer 3: 가사 자막
Layer 4: 효과/오버레이 (빛, 파티클, 안개)
Layer 5: 인트로/아웃트로 텍스트
```

### 카메라 효과 (Ken Burns)
- **Verse**: 느린 줌인 (1.0x → 1.15x, 15초)
- **Chorus**: 느린 줌아웃 (1.15x → 1.0x) or 패닝
- **Bridge**: 정지 or 빠른 줌
- **Outro**: 느린 줌아웃 + 페이드

### 이미지 전환 효과
- **기본**: 디졸브 (0.5초)
- **감정 전환**: 화이트 페이드 (희망) / 블랙 페이드 (절망)
- **클라이막스**: 글리치 or 플래시
- **여운**: 슬로우 디졸브 (1.5초)

### 자막 스타일

#### 기본 설정
- **폰트**: Pretendard Bold (한글) / Montserrat Bold (영문)
- **크기**: 제목 80pt / 가사 48pt / 부제 36pt
- **위치**: 하단 1/3 (safe zone)
- **배경**: 반투명 검정 그라데이션

#### 시리즈별 자막 색상
| 시리즈 | 메인 색 | 강조 색 |
|--------|---------|---------|
| BTS 답가 | 흰색 | 보라 (#9B59B6) |
| 번안형 | 흰색 | 문화별 컬러 |
| 팬픽OST | 흰색 | 세계관 컬러 |

#### 자막 애니메이션
- **일반 가사**: 페이드인 (0.3초)
- **핵심 가사**: 타이핑 효과
- **감정 폭발**: 스케일업 + 글로우
- **속삭임**: 작은 폰트 + 느린 페이드

### 인트로 템플릿 (쇼츠 첫 1~3초)
```
[배경: 강렬한 이미지]
[텍스트 애니메이션: 타이핑 or 글리치]
"BTS가 'SWIM'으로 돌아왔을 때"
"3년 동안 숨 참던 아미가 부르는 답가"
[0.5초 비트드롭과 함께 시작]
```

### 아웃트로 템플릿
```
[페이드아웃 중]
채널명: Re:Version Music
시리즈명: BTS ARIRANG 답가 시리즈
"구독과 좋아요 부탁드려요"
[2초]
```

## 4단계: 쇼츠 vs 롱폼

### 쇼츠 (45~60초) — 메인 콘텐츠
```
0~1초:  강한 훅 (질문형 텍스트)
1~3초:  비트드롭 + 상황 설명
3~20초: Verse + Pre-Chorus 하이라이트
20~40초: Chorus (핵심)
40~50초: Bridge 한 줄 or Final Chorus 킬링파트
50~55초: 여운 + 채널명
55~60초: CTA (구독/좋아요)
```

### 편집 포인트
- **후렴이 20초 안에 나와야 함** (이탈 방지)
- 텍스트는 **2줄 이내** 유지
- 비트에 맞춰 장면 전환
- **음악 시작 전 1~2초 텍스트만** (훅)

### 롱폼 (3~5분) — 풀버전
- 곡 전체 + 가사 자막
- 인트로에 원곡 상황 설명 (15초)
- 아웃트로에 제작 비하인드 or 다음 예고 (15초)
- 설명란에 원곡 정보, 답가 컨셉, 가사 전문

## 5단계: 업로드 체크리스트

### YouTube Shorts
- [ ] 제목: 쇼츠 훅과 동일 (60자 이내)
- [ ] 설명: 원곡 정보 + 답가 컨셉 + 가사 일부 + 해시태그
- [ ] 해시태그: #BTS #ARIRANG #SWIM #답가 #ReVersionMusic
- [ ] 썸네일: 자동 (쇼츠)
- [ ] 예약: 한국 시간 저녁 7~9시

### YouTube 롱폼
- [ ] 제목: "곡명 (원곡 답가) | Re:Version Music"
- [ ] 설명: 풀 가사 + 원곡 링크 + 제작 크레딧
- [ ] 썸네일: 커스텀 (1280×720)
- [ ] 태그: 원곡명, 아티스트명, 답가, 장르
- [ ] 챕터: Intro / Verse / Chorus / Bridge / Outro

### TikTok
- [ ] 15~60초 편집본 (쇼츠와 동일 or 후렴만)
- [ ] 캡션: 훅 + 해시태그
- [ ] 사운드: 오리지널 사운드로 등록

## 퀄리티 체크리스트

### 음악
- [ ] 후렴이 귀에 꽂히는가
- [ ] 4분 전체 구성이 지루하지 않은가
- [ ] 보컬이 선명한가
- [ ] 원곡과 멜로디가 유사하지 않은가 (저작권)

### 영상
- [ ] 첫 1초에 시선을 잡는가
- [ ] 자막이 읽기 편한가
- [ ] 이미지 전환이 비트에 맞는가
- [ ] 감정 흐름이 음악과 일치하는가
- [ ] 화질이 1080p 이상인가

### 콘텐츠
- [ ] 쇼츠 훅이 클릭을 유도하는가
- [ ] 답가 컨셉이 명확한가
- [ ] 원곡 팬이 공감할 수 있는가
- [ ] 댓글 유도 요소가 있는가
