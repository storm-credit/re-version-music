# 곡별 프로덕션 바이블 표준

## 1. 왜 필요한가

Re:Version Music은 노래만 만드는 채널이 아니라, 감정 번안 오리지널 곡을 시각 세계와 함께 완성하는 채널이다. 곡별로 캐릭터/화자 시트와 스토리보드가 없으면 영상 생성 단계에서 인물, 색감, 감정, 컷 연결이 흔들린다.

따라서 모든 주요 곡은 `Complete Production Bible`을 가진다.

## 2. 기본 구조

```text
0부. 제작 개요
1부. 캐릭터/화자 시트
2부. 음악 프롬프트
3부. 스토리보드 이미지 프롬프트
4부. 비디오 프롬프트
5부. 편집 가이드
6부. 발행 패키지
```

## 3. 포맷

기본형:

```text
9 CUT x 15초 = 2:15
16:9 롱폼 MV
영상 생성물은 SFX only
Suno 음악은 편집 단계에서 합성
```

곡 길이가 다르면 아래처럼 조정한다.

| 곡 길이 | 권장 컷 |
|---|---|
| 1:30 이하 | 6 CUT x 15초 |
| 2:00-2:30 | 8-10 CUT x 15초 |
| 3:00 이상 | 10-12 CUT 또는 20초 컷 혼합 |

## 4. 캐릭터/화자 시트

팬픽OST형, 오리지널형, K-판타지형은 캐릭터 시트가 필수다.

번안형, 답가형은 최소한 화자 시트를 만든다.

```text
이름/코드:
역할:
감정 기능:
핵심 상처:
말하지 못한 문장:
외형/소품:
컬러:
보컬 톤:
MV 반복 장면:
금지 요소:
```

## 5. 스토리보드 시트

각 CUT는 표준 표를 가진다.

| 컬럼 | 내용 |
|---|---|
| SHOT / TIME | 샷 번호와 초 단위 시간 |
| START FRAME | 시작 이미지 |
| END FRAME | 끝 이미지 |
| CAMERA / MOVEMENT | 카메라 |
| ACTION / DIRECTION | 장면 지시 |
| DIALOGUE / LYRICS | 가사/무대사 |
| SFX | 영상 자체 효과음 |
| MUSIC | 음악 구조/싱크 |

각 CUT 끝에는 아래를 붙인다.

```text
DIRECTOR'S INTENT:
TRANSITION:
OUTPUT RULES:
```

## 6. 비디오 프롬프트

비디오 프롬프트는 스토리보드와 별도다. 한 컷 15초를 생성할 때 쓴다.

```text
CUT 번호와 시간
스타일
컬러
샷별 타임코드
캐릭터 고정 묘사
오디오: SFX only
Negative
Output
```

## 7. 편집 가이드

반드시 적는다.

```text
음악 길이:
영상 길이:
드롭:
클라이맥스:
아웃트로:
SFX 볼륨:
음악 볼륨:
렌더링:
최종 체크리스트:
```

## 8. 오케스트라 판단

곡별 바이블은 아래 기준을 통과해야 한다.

- 캐릭터/화자 시트가 먼저 잠겼다.
- 스토리보드 컷이 음악 구조와 맞는다.
- 영상 프롬프트에는 음악을 넣지 않고 SFX만 남겼다.
- Suno 음악은 별도 레이어로 합성한다.
- 원곡/IP/참조곡 복제 위험이 없다.
- 곡이 먼저 좋고, 영상은 그 곡을 돕는다.
