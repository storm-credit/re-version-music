# Suno 전문가 노래 우선 게이트

## 0. 원칙

Suno 프롬프트는 멋진 표현을 많이 넣는 문서가 아니라, 생성 모델을 좁고 안전한 방향으로 몰아가는 제작 지시서다.

Re:Version Music의 Suno 단계는 아래 순서를 반드시 따른다.

1. `Title / Style of Music / Lyrics` 세 블록만 복사한다.
2. Style은 5-10개 핵심 키워드로 제한한다.
3. 원곡명, 아티스트명, `cover`, `remix`, `Korean version`, `in the style of`는 넣지 않는다.
4. 좋은 노래 검증은 최소 3개 테이크로 한다.
5. 18점 미만은 재생성, 22점 이상만 우선 후보로 둔다.

## 1. Take 전략

좋은 곡이 목표이면 한 프롬프트를 반복하는 것으로 끝내지 않는다. 장르/보컬/편곡 곡선을 나누어 최소 A/B/C 테이크를 둔다.

| Take | 목적 | 사용 상황 |
|---|---|---|
| Take A | 가장 안전한 메인 장르 | 곡의 정체성을 확인하는 1차 생성 |
| Take B | 보컬/공간감을 바꾼 절제 버전 | A가 과하거나 후렴이 늦을 때 |
| Take C | 후렴 임팩트를 키운 버전 | A/B가 밋밋하거나 반복 청취성이 약할 때 |

장르 실험은 한 번에 섞지 않는다. 한 테이크에는 중심 장르 하나만 둔다.

## 2. 생성본 선별표

각 생성본은 0-3점으로 평가한다.

| 항목 | 질문 |
|---|---|
| 후렴 훅 | 첫 줄이 제목처럼 남는가 |
| 멜로디 독립성 | 참조곡 없이 새 곡처럼 들리는가 |
| 보컬 감정 | 화자의 감정과 거리감이 맞는가 |
| 발음 | 자막 없이 핵심 줄이 들리는가 |
| 편곡 곡선 | 마지막 후렴까지 커지는가 |
| 장르 적합성 | 장르가 감정을 살리는가 |
| 반복 청취 | 다시 듣고 싶은가 |
| MV 적합성 | 장면이 바로 떠오르는가 |

판정:

| 점수 | 판단 |
|---:|---|
| 0-17 | 재생성 |
| 18-21 | 조건부 후보 |
| 22-24 | 우선 후보 |

즉시 탈락:

- 후렴 첫 진입이 너무 늦어 롱폼 초반 이탈을 유발한다.
- 핵심 후렴 첫 줄이 안 들린다.
- 참조곡/아티스트가 떠오를 정도로 유사하다.
- 보컬 감정이 화자와 맞지 않는다.
- 장르가 곡의 상처보다 장식처럼 들린다.

## 3. 모든 곡의 Suno 산출물 형식

```text
Suno Run Goal:
Title:
Take A Style:
Take B Style:
Take C Style:
Lyrics:
Avoid:
Generation Count:
Immediate Reject Rules:
Selection Checklist:
Next Action:
```

## 4. 총괄 연결

오케스트라가 구조상 승인한 곡이라도 Suno 전문가 점수 18점 미만이면 제작 보류다.

Production Bible 확장은 22점 이상 우선 후보가 나온 뒤 진행한다. 18-21점 후보는 영상/캐릭터시트를 만들 수 있지만, 발행 확정은 금지한다.
