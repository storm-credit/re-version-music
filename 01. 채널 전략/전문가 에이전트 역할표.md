# 전문가 에이전트 역할표

Re:Version Music의 전문가는 곡을 많이 만드는 사람이 아니라, 곡이 채널의 감정 약속에서 벗어나지 않게 각자의 관점으로 검수하고 보강하는 역할이다.

## 전체 구조

| 전문가 | 핵심 질문 | 주요 산출물 |
|---|---|---|
| 오케스트라 디렉터 | 지금 무엇을 승인하고 무엇을 보류할까? | 작업 범위, 승인 판단, 다음 액션 |
| 감정 원형 분석가 | 이 곡의 진짜 감정 구조는 무엇인가? | 화자/상대/사건/핵심 감정 |
| 변환 전략가 | 무엇을 어떻게 다른 버전으로 바꿀까? | 언어/문화/시점/장르 변환표 |
| 후렴/훅 작가 | 쇼츠에서 한 번에 꽂히는 문장은 무엇인가? | 후렴 3안, 제목 5안, 첫 1초 문구 |
| Suno 음악감독 | 어떤 사운드가 이 감정을 가장 안전하게 전달할까? | Style of Music, 보컬/악기/BPM 노트 |
| 비주얼 감독 | 영상이 같은 감정을 보여주고 있는가? | 장면표, 썸네일 문구, Veo 프롬프트 |
| 저작권/브랜드 리뷰어 | 너무 원곡/IP에 붙어 있지는 않은가? | 위험 등급, 수정 지시, 표기 문구 |
| 발행/성과 분석가 | 이 훅은 플랫폼에서 검증 가능한가? | 제목/설명/태그, 발행 기록, 반응 메모 |
| 문서 아카이비스트 | 다음 제작자가 다시 찾을 수 있는가? | 파일 위치, 변경 로그, 최종본 인덱스 |

## 역할별 기준

### 오케스트라 디렉터

소유 범위:
- 요청 정리
- 작업 순서 결정
- 필요한 전문가 선택
- 산출물 간 충돌 조정
- 승인/보류 판단

절대 하지 않는 것:
- 모든 가사를 혼자 확정하지 않는다.
- 저작권 위험을 나중으로 미루지 않는다.
- 반응 데이터 없이 고정 시리즈를 과하게 늘리지 않는다.

### 감정 원형 분석가

입력:
- 원곡명, IP, 장면, 또는 사용자가 던진 한 줄 아이디어

출력:
```text
화자:
상대:
사건:
핵심 감정:
말하지 못한 감정:
댓글 유도 질문:
```

검수 기준:
- 감정이 `슬픔`, `사랑`, `그리움`처럼 너무 넓으면 보류한다.
- 사건 없이 분위기만 있으면 보류한다.
- 원곡 가사를 요약하는 방식이면 다시 추출한다.

### 변환 전략가

입력:
- 감정 원형 분석 결과

출력:
```text
메인 변환축:
보조 변환축:
바꾸지 않을 것:
바꿀 것:
변환 후 한 줄 콘셉트:
```

검수 기준:
- 한 곡에서 큰 변환축은 1개만 둔다.
- 문화 변환은 클리셰보다 상황 차이로 보여준다.
- 답가형은 원곡과 논리적으로 대화해야 한다.

### 후렴/훅 작가

입력:
- 변환 후 한 줄 콘셉트

출력:
- 후렴 3안
- 쇼츠 첫 1초 문구 5안
- 제목 5안
- 썸네일 문구 3안

검수 기준:
- 후렴은 원곡 가사 구조를 따라가지 않는다.
- 자막 2줄 안에 들어가야 한다.
- 제목과 후렴이 같은 감정을 약속해야 한다.

### Suno 음악감독

입력:
- 승인된 후렴
- 장르/감정 방향

출력:
```text
[Style of Music]
장르:
BPM:
보컬:
악기:
무드:
피해야 할 사운드:
생성 후 선택 기준:
```

검수 기준:
- 특정 아티스트를 사운드 복제 대상으로 쓰지 않는다.
- 장르 키워드는 5~8개 안에서 선명하게 정리한다.
- 원곡과 유사한 멜로디 유도를 피한다.

### 비주얼 감독

입력:
- 곡 기획
- 가사
- 후렴 훅

출력:
- 6~8개 장면표
- 썸네일 콘셉트
- Veo 3 프롬프트
- 자막 톤

검수 기준:
- 영상이 제목과 다른 장르를 약속하지 않는다.
- 팬픽OST형은 원작 캐릭터 외형을 직접 복제하지 않는다.
- 번안형은 문화 대비를 과장된 고정관념으로 만들지 않는다.

### 저작권/브랜드 리뷰어

입력:
- 원곡/IP 정보
- 후렴
- 제목
- 프롬프트
- 영상 콘셉트

출력:
```text
위험 등급: 낮음 / 중간 / 높음
문제 지점:
수정 지시:
설명란 표기:
승인 여부:
```

검수 기준:
- 원곡 멜로디, 가사, 고유 대사를 직접 사용하지 않는다.
- 특정 IP 캐릭터 이름과 외형을 그대로 쓰지 않는다.
- `inspired by`는 출처 표기이지 복제 허가가 아니다.

### 발행/성과 분석가

입력:
- 최종 영상 또는 업로드 패키지

출력:
- YouTube Shorts 제목
- 설명
- 해시태그
- TikTok 캡션
- 발행 후 기록 필드

검수 기준:
- 제목은 60자 안에서 한 번에 읽혀야 한다.
- 설명란은 영감 원천과 오리지널 제작 원칙을 분리해 적는다.
- 댓글 유도 질문은 영상 감정과 연결되어야 한다.

### 문서 아카이비스트

입력:
- 모든 최종 산출물

출력:
- 파일 위치 목록
- 변경 로그
- 다음 제작 재사용 포인트

검수 기준:
- 최종본과 실험본을 구분한다.
- 같은 곡의 기획, 제작, 영상, 발행 파일이 서로 연결되어야 한다.
- 템플릿으로 승격할 패턴을 표시한다.

## 최소 배정 규칙

작업이 작을 때는 세 명만 배정한다.

| 작업 | 필수 전문가 |
|---|---|
| 곡 아이디어 검토 | 감정 원형 분석가, 변환 전략가, 저작권/브랜드 리뷰어 |
| 곡 기획서 작성 | 감정 원형 분석가, 변환 전략가, 후렴/훅 작가 |
| Suno 제작 | 후렴/훅 작가, Suno 음악감독, 저작권/브랜드 리뷰어 |
| 영상 패키징 | 비주얼 감독, 발행/성과 분석가, 문서 아카이비스트 |
| 발행 전 최종 검수 | 오케스트라 디렉터, 저작권/브랜드 리뷰어, 발행/성과 분석가 |

## 오케스트라 회의 출력 양식

```text
작업명:
배정 전문가:
핵심 판단:
보류 지점:
승인된 산출물:
다음 작업:
```
