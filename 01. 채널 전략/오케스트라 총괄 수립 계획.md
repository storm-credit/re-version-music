# 오케스트라 총괄 수립 계획

수립 일자: 2026-05-10

## 목적

Re:Version Music의 오케스트라 총괄 체계를 문서 모음이 아니라 실제 제작 운영 방식으로 정착시킨다. 목표는 `아이디어 -> 감정 원형 -> 변환축 -> 후렴 -> Suno -> 영상 -> 발행 -> 성과 기록`이 매번 같은 기준으로 통과되게 만드는 것이다.

## 수립 원칙

1. 오케스트라는 창작자를 대체하지 않는다. 판단 순서를 만든다.
2. 전문가는 더 많은 의견을 내기보다, 자기 영역의 실패를 조기에 막는다.
3. 모든 곡은 후렴 승인 전까지 풀 제작으로 넘어가지 않는다.
4. 공개 발행물은 저작권/브랜드 리뷰를 통과해야 한다.
5. 발행 후 데이터는 다음 곡의 기획에 반드시 돌아온다.

### Phase 2. 30일 운영

목표: 최소 3곡을 같은 방식으로 통과시켜 반복 가능한 리듬을 만든다.

대상:
1. 번안형: `새벽 세 시의 고백`
2. 답가형: BTS 외 일반 답가 1곡
3. 팬픽OST형: 캐릭터 아키타입 정리 후 1곡

작업:
- 매주 1곡을 오케스트라 회의록으로 통과시킨다.

- 발행 전 저작권 위험도를 기록한다.
- 결과가 나온 곡은 발행 성과 기록 템플릿에 1일/3일/7일 지표를 남긴다.

완료 기준:

- 발행 패키지 3개

### Phase 3. 90일 확장

목표: 콘텐츠 유형별 승률을 비교하고 비율을 조정한다.

측정:
- 번안형 유지율
- 답가형 댓글률
- 팬픽OST형 저장/공유 가능성
- 오리지널형 실험 반응

결정:
- 반응 좋은 유형의 비율을 10~20% 높인다.
- 반응 약한 유형은 훅, 후렴, 영상 중 어느 지점이 약했는지 분리한다.
- 성공한 후렴 구조는 템플릿으로 승격한다.

완료 기준:
- 10곡 이상 오케스트라 기록
- 유형별 성과 비교표
- 다음 30일 제작 우선순위
- 템플릿 업데이트 3개 이상

## 첫 파일럿 권장안

파일럿 곡: `새벽 세 시의 고백`

이유:
- 변환축이 선명하다.
- 원곡 의존도를 낮춰도 감정이 산다.
- 영상 제작 난도가 낮다.
- 후렴 첫 줄이 이미 강하다.

파일럿 목표:
```text

```

## 총괄 완료 정의

오케스트라 수립은 문서가 생겼다고 끝나지 않는다. 아래 조건을 만족하면 1차 수립 완료로 본다.

- 실제 곡 3개가 같은 회의록 템플릿을 통과했다.
- 발행된 곡 1개 이상이 성과 기록에 들어갔다.
- 실패한 훅과 성공한 훅이 분리 기록되었다.
- 전문가별 수정 지시가 다음 곡에 반영되었다.
- 총괄자가 다음 30일 우선순위를 숫자로 정했다.
