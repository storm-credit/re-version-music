# Phase 2 오케스트라 실행 계획

## 0. 총괄 판단

판정: 조건부 승인.

이유: Phase 1 파일럿 `새벽 세 시의 고백`은 문서상 승인됐지만, 실제 Suno 생성본 선별이 아직 비어 있다. 노래 우선 원칙상 Phase 2는 새 곡을 늘리는 단계가 아니라, 파일럿 곡을 실제 음원/영상/발행 데이터까지 닫고 그 방식으로 다음 2곡을 통과시키는 단계다. 또한 전체 후보는 `ReVersion 구조 감사 - 노래 우선 운영안`의 대화 구조/노래 우선 게이트를 통과해야 한다.

## 1. 전문가 보고 요약

| 전문가 | 판정 | 핵심 결정 |
|---|---|---|
| 좋은 곡 모티브/작곡 전략가 | 조건부 승인 | Take A 3회 후 A가 과하면 Take B, 약하면 Take C로 이동 |
| 가사 디렉터 + Suno 음악감독 | 조건부 승인 | Take A 생성 가능. 18-21점은 조건부 후보, 22점 이상 우선 후보 |
| 캐릭터시트 + MV 디렉터 | 조건부 승인 | DAWN-03 화자 시트는 잠금 가능. Haechi식 CUT/SHOT 표는 아직 필요 |
| 저작권/브랜드 리뷰어 | 조건부 승인 | 공개 제목 C와 `직진 이별` 프레이밍은 공개 영역에서 제외 |
| 웹 플랫폼 아키텍트 | 조건부 승인 | Production Bible Builder MVP가 Phase 2 첫 구현 대상 |
| 문서 아카이비스트 + 운영 PM | 조건부 승인 | 30일 운영은 파일럿 발행 완료 -> 답가형 -> 팬픽OST형 순서 |

2026-05-15 추가 통합 판단:

- Suno 전문가: Take A x3 시작 승인. 실패 시 무조건 C가 아니라 A가 과하면 B, 약하면 C.
- 답가형 작사/구조 전문가: `사실 못 보냈던 말`은 화자/보컬/가사 시점 충돌로 제작 보류.
- MV/Production Bible 전문가: 확정 음원 전에도 캐릭터시트 확장과 10-12 CUT 구조 초안은 진행 가능.
- 안전 전문가: 답가형 1/2는 공개 훅에서 원곡명/아티스트명 제거 전까지 공개 보류.
- 운영 PM: 2026-05-21까지 파일럿 음원 후보 확정이 1순위.

## 3. Week 1 작업 지시

대상 곡: `새벽 세 시의 고백`

선행 기준: 전체 후보의 구조 감사는 `01. 채널 전략/ReVersion 구조 감사 - 노래 우선 운영안.md`를 기준으로 진행한다. Suno 생성/선별은 `01. 채널 전략/Suno 전문가 노래 우선 게이트.md`를 따른다.

필수 액션:

1. Suno Take A 3회 생성.
2. `10. Suno 생성 세션 로그.md`에 길이, 후렴 첫 진입, 즉시 탈락 여부 기록.
3. `07. 생성본 선별 체크리스트.md`에 8항목 점수 기록.
4. 22점 이상이면 최종 후보, 18-21점이면 조건부 후보.
5. 18점 이상 후보가 없으면 Suno 전문가 기준으로 Take B 또는 Take C 2회로 이동.
6. 최종 후보가 생긴 뒤 롱폼 영상 타임코드 확정.
7. 기존 Scene 1-10을 `10 CUT x 15초` 또는 실제 곡 길이에 맞춘 CUT/SHOT 스토리보드 표로 변환.

## 4. Week 2 대상

대상 후보: `02. 곡 기획/답가형/01. 이별 통보에 대한 남겨진 사람의 답가.md`

운영 원칙:

- 공개 제목에는 원곡명/아티스트명을 쓰지 않는다.
- 감정 원형은 `떠난 사람에게 차마 못 보낸 말`로 재정의한다.
- 후렴 3안, 제목 3안, 썸네일 문구 3안을 반드시 남긴다.
- 기존 기획을 그대로 쓰지 않고 오케스트라 인테이크/회의록/승인 매트릭스로 재포맷한다.

## 5. Week 3-4 대상

대상 후보: `02. 곡 기획/팬픽OST형/01. 회귀한 악녀의 마지막 선택.md`

운영 원칙:

- 특정 웹툰명/원작명/고유 장면 의존을 공개 패키지에서 제거한다.
- `회귀 악녀 장르 전반의 아키타입 OST`로 재정의한다.
- 캐릭터시트 디렉터가 먼저 화자/상대/소품/금지 IP 요소를 잠근다.
- 이후 Suno/MV/Production Bible로 넘어간다.

## 6. 웹 플랫폼 실행

Phase 2 첫 구현 대상: Production Bible Builder MVP.

범위:

- Generate Bible Draft
- Parse Cuts
- Export Markdown

보류:

- PostgreSQL 전환
- 원격 호스팅
- 렌더 대시보드 선구현

강제 조건:

- 캐릭터/화자 시트 승인 전 스토리보드 생성 금지.
- 안전 리뷰 전 발행 패키지 승인 금지.
- prompt provenance 없는 생성물 승인 금지.
